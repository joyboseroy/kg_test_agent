# KG-Guided Generative Test Agent

Companion code for the paper:

**Context-Aware Generative AI for Automated Telecom Test Script Generation**
*(Planned for arXiv — citation will be updated on publication)*

---

## What this is

A prototype implementation of the KG-guided generative test generation
framework described in the paper. The system:

1. Ingests a Python KPI monitoring application into a **FalkorDB knowledge graph**
2. Runs a **delta engine** to detect features with no test coverage
3. Exposes KG tools via an **MCP server**
4. Uses a **TinyLlama agent** (via Ollama) to generate pytest test cases
5. Writes generated tests back to the KG — closing the feedback loop

---

## Architecture

```
kpi_app/kpi.py              Target system: 6 KPI features (3 initial + 3 new)
        |
        v
kg/ingest.py                Parse Python AST -> populate FalkorDB KG
        |
        v
FalkorDB                    Knowledge graph: File, Function, Feature, TestCase nodes
        |
        v
kg/query.py                 Cypher query helpers over FalkorDB
        |
        v
kg/delta.py                 Delta engine: find features with no test coverage
        |
        v
mcp_server/server.py        MCP server: exposes KG tools to the agent
        |
        v
agent/agent.py              TinyLlama agent: queries KG, generates tests, writes back
        |
        v
tests/test_generated_kpi.py Generated test file
```

## Setup

### 1. Start FalkorDB

```bash
docker-compose up -d
```

Or directly:
```bash
docker run -p 6379:6379 falkordb/falkordb
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Install and start Ollama + TinyLlama

```bash
# Install Ollama from https://ollama.com
ollama pull tinyllama
ollama serve
```

## Demo walkthrough

Replicates the demonstration in paper Section 5.1 and 6.3.

### Step 1 — Ingest initial state (3 features)

```bash
python kg/ingest.py --mode initial
```

Output:
```
Ingested: calculate_cell_availability -> Feature: cell_availability [initial]
Ingested: check_rsrp_signal -> Feature: rsrp_signal_quality [initial]
Ingested: check_call_drop_rate -> Feature: call_drop_rate [initial]

Knowledge Graph summary:
  Feature: 3 nodes
  File: 1 nodes
  Function: 3 nodes
```

### Step 2 — Verify initial tests pass

```bash
pytest tests/test_kpi_initial.py -v
```

22 tests, all passing.

### Step 3 — Simulate feature branch (ingest 3 new features)

```bash
python kg/ingest.py --mode extended
```

Adds 3 new Feature nodes with status="new" and no test coverage.

### Step 4 — Run delta engine

```bash
python kg/delta.py
```

Output:
```
3 of 6 features have no test coverage. Coverage: 50%.
3 high-priority gaps (new features).

Coverage gaps (3):
  [HIGH] handover_success_rate
  [HIGH] latency_sla
  [HIGH] network_throughput
```

### Step 5 — Start MCP server (separate terminal)

```bash
python mcp_server/server.py
```

### Step 6 — Run the agent

```bash
python agent/agent.py --output tests/test_generated_kpi.py
```

The agent queries KG for coverage gaps, builds prompts with function
signatures and docstrings, calls TinyLlama for each gap, writes generated
tests to file, and persists test metadata back to FalkorDB.

### Step 7 — Verify generated tests

```bash
pytest tests/test_generated_kpi.py -v
```

### Step 8 — Check updated KG coverage

```bash
python kg/delta.py
```

All 6 features now show as COVERED.

## KG node and edge types

```
(File)-[:CONTAINS]->(Function)-[:IMPLEMENTS]->(Feature)-[:TESTED_BY]->(TestCase)
                                                                          |
                                              (TestCase)-[:COVERS]->(Function)
```

Queried via Cypher through FalkorDB Python client.

---

## MCP tools

| Tool | Description |
|---|---|
| `query_kg_coverage` | Get feature coverage summary |
| `get_coverage_gaps` | Get uncovered features with function context |
| `get_delta_report` | Full delta report for agent |
| `get_feature_context` | Detailed context for one feature |
| `write_testcase_to_kg` | Persist generated test back to KG |

---

## Note on TinyLlama

TinyLlama is a 1.1B parameter model. Test quality will be basic compared
to larger models. The framework is model-agnostic:

```bash
OLLAMA_MODEL=llama3.2 python agent/agent.py
OLLAMA_MODEL=mistral python agent/agent.py
```
